Étape 3 — Lire l'email reçu avec l'API Boomlify
Boomlify permet de lire les emails reçus via son API. On interroge l'endpoint en boucle jusqu'à recevoir l'email.
// Interroge Boomlify toutes les 3s jusqu'à réception for (let i = 0; i < 10; i++) { const res = await fetch( `https://v1.boomlify.com/api/emails/${inboxId}`, { headers: { 'x-api-key': boomlifyKey } } ); const data = await res.json(); const emails = data.data || data.emails || []; if (emails.length > 0) return emails[0]; // ✓ reçu await new Promise(r => setTimeout(r, 3000)); }
Étape 4 — Extraire le lien avec une regex
Une expression régulière (regex) cherche dans le HTML de l'email un lien de confirmation.
// Cherche un lien contenant confirm/verify/activate/token const regex = /href=["'](https?:\/\/[^"']*(?:confirm|verify|activate|token)[^"']*)/i; const match = emailBody.match(regex); const link = match ? match[1] : null; // → "https://finder-niche.com/confirm?token=abc123"
Étape 5 — ScraperBee visite le lien et récupère la clé
ScraperBee ouvre la page de confirmation. Cette page affiche directement la clé API — le script JS la détecte automatiquement dans les input, span ou code de la page.
// ScraperBee visite le lien ET extrait la clé en même temps const result = await fetch('https://app.scrapingbee.com/api/v1/', { body: JSON.stringify({ api_key: scraperBeeKey, url: confirmLink, // Le lien de l'email render_js: true, wait: 3000, js_scenario: { instructions: [ { wait: 2000 }, // Extraire la clé API depuis la page { evaluate: SCRIPT_GET_KEY } ] } }) }); // result.js_return_value = la clé API !
